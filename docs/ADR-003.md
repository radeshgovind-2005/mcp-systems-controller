# ADR 003: Infrastructure API Integration and Data Abstraction

**Date:** 2026-03-27
**Status:** Accepted

## Context
The system requires direct, secure communication with the Docker Daemon and the Kubernetes API to execute infrastructure commands. Additionally, passing raw API responses (which are often massive, deeply nested JSON objects) directly to an LLM agent consumes excessive context window tokens and drastically increases the likelihood of the LLM hallucinating parameters.

## Decision
1. **Docker Client:** We selected `docker-java` utilizing the `ApacheDockerHttpClient` transport for stable, connection-pooled HTTP communication with the Docker socket.
2. **Kubernetes Client:** We selected the official `kubernetes-client/java`, relying on `Config.defaultClient()` to seamlessly adapt to the local `~/.kube/config` or in-cluster service accounts.
3. **Data Abstraction:** We implemented a strict abstraction layer (`DockerTools.kt` and `KubernetesTools.kt`). Instead of returning raw API models to the agent, these services parse the responses and return simplified Kotlin `List<Map<String, String>>` or plain strings.

## Rationale
* **Token Efficiency & Reliability:** By distilling a 500-line raw Kubernetes Pod JSON down to just `name`, `status`, and `podIP`, we save thousands of tokens per MCP interaction. This forces the agent to rely on deterministic, curated data rather than guessing which JSON fields matter.
* **Synchronous Log Aggregation:** `docker-java` streams logs asynchronously via callbacks. We wrapped this in a synchronous `awaitCompletion` block with a timeout. LLM agents operate on a request-response cycle; they cannot process continuous asynchronous streams, so we must buffer and return a discrete chunk of logs.

## Consequences
* **Positive:** Drastically reduced token usage and faster LLM response times.
* **Positive:** The system is agnostic to *how* it's deployed (works locally with Minikube/Docker Desktop, and works in production clusters).
* **Negative:** Adding new fields for the agent to read requires manually updating the data mappers in the tool services.
