# ADR 001: Foundational Stack and Core Domain Modeling

**Date:** 2026-03-27
**Status:** Accepted

## Context
The `mcp-systems-controller` acts as a secure bridge between non-deterministic LLM agents and highly deterministic infrastructure environments (Docker, Kubernetes). The system must handle requests via the Model Context Protocol (MCP), execute read-only operations immediately, and safely suspend destructive operations until a human operator provides explicit, asynchronous approval. 

To achieve this, the system requires:
1. Strict type safety to prevent malformed infrastructure commands.
2. Robust, non-blocking concurrency to hold pending `PlanOfAction` (PoA) requests without exhausting server threads.
3. Native observability for audit trails and latency tracking.

## Decision
We have selected the following foundational stack and domain architecture:

* **Language:** Kotlin (JVM). 
* **Framework:** Spring Boot 3.4.x.
* **Build System:** Gradle (Kotlin DSL).
* **Concurrency Model:** Kotlin Coroutines.
* **Core Infrastructure Clients:** `docker-java` (Docker) and `kubernetes-client` (Official K8s API).
* **Protocol Bridge:** `spring-ai-mcp` (Spring's experimental MCP SDK).
* **Domain Design:** We established a strict state machine for approvals using an `ApprovalStatus` enum (`PENDING`, `APPROVED`, `REJECTED`, `EXECUTED`, `FAILED`) and a central `PlanOfAction` data class to encapsulate the agent's intent.

## Rationale
* **Why Kotlin over Python?** Python is the standard for AI/ML, but this is an *infrastructure controller*. A mistyped dictionary key in Python causing a runtime error during a Kubernetes deployment is unacceptable. Kotlin provides compile-time safety, null safety, and excellent tooling for enterprise infrastructure.
* **Why Spring Boot?** It provides out-of-the-box, production-ready observability (Actuator/Micrometer) and dependency injection, which accelerates the setup of our Prometheus metrics (`/actuator/prometheus`).
* **Why Kotlin Coroutines?** Holding a thread open while waiting for a Human-in-the-Loop (HITL) approval is a massive bottleneck. Coroutines allow us to suspend the execution state cheaply, freeing up the underlying thread, and natively supporting the Actor model we will use for the Audit Log to prevent concurrent write issues.

## Consequences
* **Positive:** The system will be highly concurrent, memory-efficient, and type-safe. Refactoring infrastructure tool definitions will be caught by the compiler.
* **Positive:** Observability and metrics are practically free with Spring Boot Actuator.
* **Negative/Trade-off:** The Java/Kotlin ecosystem for MCP is slightly less mature than the TypeScript/Python SDKs, meaning we will rely on Spring AI's experimental MCP modules.
