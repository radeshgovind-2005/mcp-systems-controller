# ADR 004: MCP Tool Exposure, Execution Routing, and Observability

**Date:** 2026-03-27
**Status:** Accepted

## Context
We needed a mechanism to expose our abstracted infrastructure tools (`DockerTools`, `KubernetesTools`) to the LLM agent using the Model Context Protocol (MCP). Furthermore, we needed to route approved destructive actions to their actual execution paths and ensure every interaction is tracked for observability.

## Decision
1. **Tool Registration:** We utilized Spring AI's `@Tool` annotation within `McpToolRegistry.kt` to automatically generate the MCP schema and expose functions to the agent.
2. **Execution Decoupling:** We introduced an `ExecutionService` separate from the `ApprovalGate` and `McpToolRegistry`.
3. **Admin Interface:** We created a standard REST controller (`AdminController.kt`) to act as the Human-in-the-Loop boundary.
4. **Telemetry:** We injected Micrometer's `MeterRegistry` directly into the tool registry to capture P95/P99 latency (`Timer`) for read operations and state transitions (`Counter`) for the HITL flow.

## Rationale
* **Separation of Concerns:** By decoupling the `ExecutionService` from the `McpToolRegistry`, the registry is only responsible for *intent* (creating a Plan of Action) and the execution service is only responsible for *action* (running the command). This prevents complex circular dependencies.
* **Native Protocol Support:** Spring AI's `@Tool` handles the serialization of our Kotlin functions into the JSON-RPC format required by MCP, saving us from writing boilerplate protocol parsers.
* **Granular Observability:** Tracking metrics directly at the `@Tool` level ensures that we capture the exact latency experienced by the LLM agent, which is a critical SLA metric in production AI systems.

## Consequences
* **Positive:** The system is highly modular. Adding a new infrastructure tool just requires a new `@Tool` function and a new `when` branch in the execution router.
* **Positive:** Complete visibility into agent behavior and human approval rates via Prometheus.
* **Negative:** The Spring AI MCP implementation is currently experimental (0.1.0-SNAPSHOT), which means protocol upgrades might require minor refactoring in the future.
